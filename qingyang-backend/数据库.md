# 为「轻养AI」添加数据库与用户认证（注册/登录 JWT）

## 设计原则

- **自底向上**：数据库配置 → 数据模型 → 核心逻辑 → API 路由 → main.py 挂载
- **标准结构**：models (ORM)、schemas (Pydantic)、crud (数据操作)、routers (路由)、utils (工具) 分离
- **每步可验证**：每一步完成后都可独立测试，能正常启动

> **为什么要把 `models/` 改成 `schemas/`？** 现有 `app/models/` 存放的是 Pydantic 模型（请求/响应结构），按标准 FastAPI 架构应归类为 `schemas/`。改名后 `models/` 留给 SQLAlchemy ORM 使用，语义清晰。已有路由需要同步改 import 路径。

#### User Schema 定义要点

```python
# schemas/user.py 需要包含以下 class（伪代码示意）

class UserCreate(BaseModel):
    email: str           # EmailStr 需要额外包，用 str + @validator 就行
    username: str        # min_length=2, max_length=50
    password: str        # min_length=6, 用 @field_validator 校验

class UserLogin(BaseModel):
    email: str
    password: str

class UserResponse(BaseModel):
    id: int
    email: str
    username: str
    created_at: datetime
    model_config = {"from_attributes": True}   # ← 支持 ORM → Pydantic

class Token(BaseModel):
    access_token: str
    token_type: str = "bearer"

class TokenData(BaseModel):
    email: str | None = None   # 内部使用，不作为 API 响应
```

#### 防坑指南

- `UserResponse` 必须有 `model_config = {"from_attributes": True}`，否则 `orm_mode` 在 Pydantic v2 已废弃
- Pydantic v2 的 validator 用 `@field_validator('password')`，不是 `@validator`
- `email` 字段用 `str` 类型 + 简单校验即可，`pydantic.EmailStr` 需要 `pip install pydantic[email]`
- 移入 schemas 后删除旧的 models 文件，防止两处都引用导致混乱

```bash

print('Schema OK:', u.model_dump())
```

### Step 4：安全工具 — `app/utils/security.py`

**目标**：实现密码哈希和 JWT 令牌的创建/验证函数。

#### 新建文件

| 文件 | 核心函数 | 说明 |
|---|---|---|
| `app/utils/__init__.py` | — | 空包 |
| `app/utils/security.py` | `hash_password()`, `verify_password()`, `create_access_token()`, `decode_access_token()` | 全局单例 CryptContext + jose |

#### 函数签名与逻辑

```python
# app/utils/security.py 结构

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def create_access_token(data: dict, expires_delta: timedelta | None = None) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(minutes=30))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

def decode_access_token(token: str) -> dict:
    return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
```

#### 防坑指南

- `pwd_context` 创建为**模块级别单例**，不要每次调用都创建
- JWT 的 `sub`（subject）字段存用户的 email，作为唯一标识
- `decode_access_token` 只负责解码，不处理异常 — 由调用方（dependencies）处理 `JWTError`
- `create_access_token` 的 `exp` 是 UTC 时间戳
- `SECRET_KEY`、`ALGORITHM`、`ACCESS_TOKEN_EXPIRE_MINUTES` 从 `settings` 读取

#### ✅ 验证方法

```bash
python -c "
from app.utils.security import hash_password, verify_password, create_access_token, decode_access_token
hp = hash_password('hello123')
assert verify_password('hello123', hp)
print('Password hashing OK')
token = create_access_token({'sub': 'a@b.com'})
payload = decode_access_token(token)
print('JWT OK, sub:', payload.get('sub'))
"
```

#### Auth Router — 两个端点

| 端点 | 请求体 | 成功响应 | 错误场景 |
|---|---|---|---|

#### main.py 启动建表

```python
# main.py
from app.db.session import engine
from app.db.base import Base
# 注意：需要 import app.models.user 确保表被注册到 Base 上

@app.on_event("startup")
def on_startup():
    Base.metadata.create_all(bind=engine)
```

#### 防坑指南

- **注册端点**：`create_user` 里一定要 hash password 后再入库，业务层而不是路由层做哈希
- **登录端点**：验证失败统一返回「邮箱或密码错误」，不要区分"邮箱不存在"和"密码错误"（安全考虑）
- **get_current_user** 依赖中 token 解码失败 → 401，用 `HTTPException(status_code=401, detail="无效的认证凭据")`
- `oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/api/auth/login")` — 这个 `tokenUrl` 让 Swagger UI 自动弹出"Authorize"按钮
- **启动建表之前**必须 import User model 文件，否则 `Base.metadata` 没有 User 表的信息，`create_all` 什么也不会创建

## 关键技术要点速查

| 主题 | 规则 |
|---|---|
| 密码安全 | `passlib[bcrypt]`，永远不存明文 |
| JWT 错误信息 | 解码失败 → 401，统一「无效的认证凭据」 |
| 登录失败 | 不区分「邮箱不存在」vs「密码错误」 |
| SQLite 多线程 | `connect_args={"check_same_thread": False}` |
| Pydantic v2 ORM | `model_config = {"from_attributes": True}` |
| UserResponse 敏感字段 | 不返回 `hashed_password`（用 `model_dump(exclude={...})`） |
| 启动建表 | 确保模型文件在 `create_all` 前已被 import |
